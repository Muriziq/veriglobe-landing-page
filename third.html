Veriglobe API Documentation

Getting Started

This documentation will help you integrate Veriglobe's identity verification services into your application. If you need any assistance, please contact our support team.

The Veriglobe API provides a comprehensive suite of identity verification services, allowing developers to integrate KYC (Know Your Customer) functionality into their applications.

Our API enables you to verify various forms of identification including Bank Verification Numbers (BVN), National Identification Numbers (NIN), identity documents, and address information.

Generate KYC Link

Create a verification session and get a link for your customers to complete verification.

Retrieve User Data

Access comprehensive verification data for your customers after they complete the process.

List Verifications

View all verification sessions for your projects and track their status. Base

Base URL

  1. https://api.veriglobe.net

API Flow

  1. Generate a KYC link for your customer and store the verification_id
  2. Customer completes verification through the provided link
  3. Receive verification results via callback or retrieve them via API using the verification_id

Authentication

All API requests require authentication using API keys. Include your API keys in the request headers:

  1. Public-Key: YOUR_PUBLIC_KEY
  2. Private-Key: YOUR_PRIVATE_KEY
Security Notice

Your API keys carry many privileges, so be sure to keep them secure! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, etc.

Security Best Practices

  • Never expose your private key in client-side code or public repositories
  • Always use HTTPS for API calls to ensure data is encrypted in transit
  • Rotate your API keysperiodically to minimize security risks
  • Implement proper error handlingfor authentication failures

Generate KYC Link

Endpoint Overview

This endpoint creates a verification session and returns a link that can be used to complete the verification process. The response includes both the verification_id and the KYC link.

index.php/generate_kyc_link
Request
Response

Headers

Content-Type

application/json

Public-Key

application/json

Private-Key

Your private API key

Request Body

project_id 

Your project ID

customer_name 

The name of the customer to verify

callback_url 

URL to receive verification results
  1. { "project_id": "YOUR_PROJECT_ID", "customer_name": "John Doe", "callback_url": "https://your-website.com/callback"}

List Verifications

Retrieves a list of verifications for a specific project. This endpoint is particularly useful for mobile applications to find the verification_id for a specific customer.

POST
index.php/list_verifications

Headers

Content-Type

application/json

Public-Key

application/json

Private-Key

Your private API key

Request Body

project_id 

The ID of the project to list verifications for

limit 

Maximum number of verifications to return (default: 100)

offset  

Number of verifications to skip (default: 0)
